Arizona Badminton Center Website

Heuristic Evaluation | User Research | Web analytics

This project is an assignment of a user experience related course, which is called Usability/ User Experience. I’ve done this project on my own. In this case study, I am going to talk about AZBC’s website. Since I like playing badminton, I visited the website several times. I found some frustration while using the website. Hence, I chose this website to do the user experience research. I will do the heuristic evaluation and user research for the website and create personas and users stories.

Challenge!

The main goal of AZBC’s website is to provide information about all ability levels of training courses, tournaments, and services, such as stringing rackets, a accessories store. However, I couldn’t find the information well on the website. I thought it was my problem. After asking some people on the badminton court, I found they all have the same issue. They couldn’t find the information they wanted on the website. Hence, I did this research on the website to try to find a solution to the issue.

About the company

Arizona Badminton Center (AZBC) is a badminton center established in 2012 in Arizona. The vision of the Center is to provide the very best badminton-specific facility for players of all ages and ability levels 7 days a week year-round. The center provides not only kids classes, intro adult classes, small group, and private lessons, tournaments of all types for all ability levels, but also a fully operational shop with stringing services and a wide array of rackets and accessories. The center’s website provides all information about open hours, class fees, schedule, and tournaments.

Heuristic Evaluation

The website, Arizona Badminton Center (http://www.azbadmintoncenter.com), will be evaluated by Jakob Nielsen's 10 principles and scaled by Jeff Sauro’s three levels rating scale.

Heuristic Evaluation - Jakob Nielsen's 10 principles

Jakob Nielsen's 10 principles are a type of heuristic evaluation for user interface design. They are called “heuristic” since they are broad rules of thumb and not specific usability guidelines(Jacob, 1994). The following list are Jakob Nielsen's 10 principles:

  1. Visibility of system status
  2. Match between system and the real world
  3. User control and freedom
  4. Consistency and standard
  5. Error prevention
  6. Recognition rather than recall
  7. Flexibility and efficiency of use
  8. Aesthetic and minimalist design
  9. Help users recognize, diagnose, and recover from errors
  10. Help and documentation

Rating Scale - Jeff Sauro’s three levels rating scale

Both problem frequency and severity are crucial when communicating the importance of usability issues. Rating the severity of the problem is less objective than finding the problem frequency. There are several ways to rate the severity. Jeff Sauro compared some popular rating approach and sorted out the following rating levels(Jeff, 2013):

  1. Minor: Cause some hesitation or slight irritation
  2. Moderate: Causes occasional task failure for some users; causes delays and moderate irritation
  3. Critical: Leads to task failure. Causes users extreme irritation

Design and Navigation Evaluation

1.     Visibility of system status -
“The design should always keep users informed about what is going on, through appropriate feedback within a reasonable amount of time.”
  • Issue: When using the searching bar at the top-right corner, the site doesn’t tell users whether it finished searching or not. It might take users some more time to see if the site finished searching.
  • Rating: Minor
  • Suggestion: Using a progress bar or showing the number of results above the searching result like Google could help users know the site finished searching.
2.     User control and freedom -
“Users often perform actions by mistake. They need a clearly marked "emergency exit" to leave the unwanted action without having to go through an extended process.”

This is a simple site, which means there are no complex links to pages. Each item in the menu links to one page. The site doesn’t have complicated forms that need undo or redo actions. Hence, users will not make mistakes in entering unwanted pages. They can control whichever page they want to enter or exit on this website.

3.     Flexibility and efficiency of use -
“Shortcuts — hidden from novice users — may speed up the interaction for the expert user such that the design can cater to both inexperienced and experienced users. Allow users to tailor frequent actions. ”
  • Issue: When contacting the website through email, users need to open their mail website, copy the site’s email address, and send the email. This might take users more time to copy and paste the site’s email address, which is something the system could do for users.
  • Rating: Minor
  • Suggestion: Although there’s a contact form on the site for users to send the email, it still needs one more click for users. Adding a hyperlink to sending email on the email address could help users directly enter their email page, which enhances the website’s flexibility and efficiency of use.
4.     Aesthetic and minimalist design -
“Interfaces should not contain information that is irrelevant or rarely needed. Every extra unit of information in an interface competes with the relevant units of information and diminishes their relative visibility. ”
  • Issue: The site doesn’t have aesthetic design. It uses different colors, font sizes, upper cases, and italics to differentiate and strengthen different content. Some of them are unnecessary, such as the text in the following image. It uses upper case letters, italics, and under-line simultaneously, which might confuse the user which content is more important. Users will not read the content, but skim it. What’s more, upper case letters are difficult to understand than lower case letters. All the above things may cost users more effort to tell the text’s level of importance.
  • Rating: Moderate
  • Suggestion: By applying one of the styles on the text, users would know which information is more important.

Content Evaluation

1.     Match between system and the real world -
“The design should speak the users' language. Use words, phrases, and concepts familiar to the user, rather than internal jargon. Follow real-world conventions, making information appear in a natural and logical order. ”

The most important information on this site is time, price, and location. And the terms that the site uses are easy to understand without any confusion. The rest of information relating to history and events is easy to understand as well. Again this is a simple website, not only the design, but also the content. The main goal of this site is to let people know news, events, open hours, and location of Arizona Badminton Center. And the words and phrases used on the site are proper for users.

2.     Consistency and standards -
“Users should not have to wonder whether different words, situations, or actions mean the same thing. Follow platform and industry conventions. ”
  • Issue: The site uses a lot of underlines to highlight the important information. However, underlined texts are usually assumed to be hyperlinks. And they are not on this website, which is violating the conventions.
  • Rating: Moderate
  • Suggestion: Avoiding using underlines to highlight text can help users not get confused with the underlined words. There are several methods to highlight text, such as capitalized words, bigger font, or different typefaces.
3.     Help and documentation -
“It’s best if the system doesn’t need any additional explanation. However, it may be necessary to provide documentation to help users understand how to complete their tasks. ”
  • Issue: In the directions section, the site provides several ways to lead users to the center by texts. The instructions are clear and detailed. However, some users who are not familiar with this area may not understand the instructions by reading the content.
  • Rating: Critical
  • Suggestion: Providing a map could definitely help and solve the problem. But a link to Google Maps would be even better. A link to Google Maps allows users to know how far is the center, how long does it take to the center, and probably the open hours, which may be more friendly to users.

Recognition and Errors Evaluation

1.     Recognition rather than recall -
“Minimize the user's memory load by making elements, actions, and options visible. The user should not have to remember information from one part of the interface to another. Information required to use the design (e.g. field labels or menu items) should be visible or easily retrievable when needed. ”
  • Issue: As we mentioned before, the most important information on the site is time, price, and location. The site shows the contact and location in the header. No matter which sections users interact with, that information is always in the header. However, the open hours are missing in the header. It may cost effort to check the open hours when users are in the Event section and want to attend the events or tournaments.
  • Rating: Moderate
  • Suggestion: Simply adding open hours in the header would solve the problem. It doesn’t have to include all the detailed information about open hours, but the regular one is good enough.
2.     Help users recognize, diagnose, and recover from errors -
“Error messages should be expressed in plain language (no error codes), precisely indicate the problem, and constructively suggest a solution. ”

Since this is a simple website, errors could only happen in search engines. The site clearly shows and tells users there’s no results found. This is good enough for users to recognize, diagnose, and recover from errors.

3.     Error prevention -
“Good error messages are important, but the best designs carefully prevent problems from occurring in the first place. Either eliminate error-prone conditions, or check for them and present users with a confirmation option before they commit to the action. ”

There are two types of errors: slips(right intent, wrong action) and mistakes(wrong intent). In this simple website, there are only three types of action: submit a form, click to sections, and search a query. Hence, a slip would only happen when submitting a form. If users fill in the wrong format in the form, the slip could happen. Hence, the system shows an instruction to prevent the error occurring, which satisfies the error prevention heuristic evaluation.

Persona Creation:

User Stories:

1.   Vivian - full time student
As a consumer, I want to check if AZBC strings rackets so that I can get my racket strung.
  • A navigation bar includes a tab links to the service page
  • A list of services that includes stringing rackets
  • A navigation bar includes a tab links to the open hours page
  • A court schedule or open hours listed
  • The location and contact of the center is shown at the bottom of the page
2.   Ariel - college student has a part time job
As a consumer, I want to check if AZBC sells rackets so that I can buy a racket.
  • A navigation bar includes a tab links to the service page
  • A list of services that includes shop store
  • A link to the shop store
  • A thumbnail image image for each product
  • Details for the product
  • A button for adding the product to the cart from detail page
  • A button for checking out the product
  • A page of order detail
  • A button links to make a payment
3.   Devanand - software engineer
As an intermediate badminton player, I want to check if AZBC holds tournaments or training classes so that I can improve my badminton skill.
  • A navigation bar includes a tab links to the event page
  • A list of upcoming tournaments
  • A navigation bar includes a tab links to the service page
  • A list of services that includes training courses
  • A link to training course page
  • A list of all courses information includes fee, time, location
  • A button to register the course
  • A form to fill out for registration
  • A button to submit the form

What I learnt

  • The personas are better to be consistent. Each of them should stick to the same style. This could make the persona more clear.
  • While designing the survey, the demographics are necessary for creating personas. Personas culture, habits, and financial condition would all affect the user’s story, their motivation, and goal.
  • Although the AZBC website is a simple one, there are still many things that need to be evaluated heuristically.
